## Overview

The GRVT platform provides multiple environments for different stages of your development and deployment lifecycle. Each environment has its own domain, chain ID, and purpose.

## Available Environments

The SDK supports four environments through the `EGrvtEnvironment` enum:

```typescript src/config/config.ts
export enum EGrvtEnvironment {
  PRODUCTION = 'PRODUCTION',
  TESTNET = 'TESTNET',
  STAGING = 'STAGING',
  DEV = 'DEV',
}
```

## Environment Details

### Production

<Note>
Use this environment for live trading with real assets.
</Note>

- **Environment**: `EGrvtEnvironment.PRODUCTION`
- **Domain**: `grvt.io`
- **Chain ID**: `325`
- **Edge URL**: `https://edge.grvt.io`

```typescript
const client = new GrvtClient({
  apiKey: process.env.GRVT_API_KEY,
  apiSecret: process.env.GRVT_API_SECRET,
  env: EGrvtEnvironment.PRODUCTION
});
```

**When to use:**
- Production applications with real users
- Live trading operations
- Real asset transfers and withdrawals

<Warning>
Always use production API keys with restricted permissions and secure storage in production environments.
</Warning>

### Testnet

<Info>
Recommended for most development and testing scenarios.
</Info>

- **Environment**: `EGrvtEnvironment.TESTNET`
- **Domain**: `testnet.grvt.io`
- **Chain ID**: `326`
- **Edge URL**: `https://edge.testnet.grvt.io`

```typescript
const client = new GrvtClient({
  apiKey: process.env.GRVT_TESTNET_API_KEY,
  apiSecret: process.env.GRVT_TESTNET_API_SECRET,
  env: EGrvtEnvironment.TESTNET
});
```

**When to use:**
- Application development and testing
- Integration testing
- User acceptance testing (UAT)
- Learning the API without risk
- Testing new features before production deployment

### Staging

- **Environment**: `EGrvtEnvironment.STAGING`
- **Domain**: `staging.gravitymarkets.io`
- **Chain ID**: `327`
- **Edge URL**: `https://edge.staging.gravitymarkets.io`

```typescript
const client = new GrvtClient({
  apiKey: process.env.GRVT_STAGING_API_KEY,
  apiSecret: process.env.GRVT_STAGING_API_SECRET,
  env: EGrvtEnvironment.STAGING
});
```

**When to use:**
- Pre-production testing
- Final validation before production release
- Testing with production-like data
- Performance testing

### Development

- **Environment**: `EGrvtEnvironment.DEV`
- **Domain**: `dev.gravitymarkets.io`
- **Chain ID**: `327`
- **Edge URL**: `https://edge.dev.gravitymarkets.io`

```typescript
const client = new GrvtClient({
  apiKey: process.env.GRVT_DEV_API_KEY,
  apiSecret: process.env.GRVT_DEV_API_SECRET,
  env: EGrvtEnvironment.DEV
});
```

**When to use:**
- Early-stage development
- Testing bleeding-edge features
- Internal development testing
- SDK development and debugging

<Warning>
The DEV environment may be unstable and is subject to frequent changes. Not recommended for external users.
</Warning>

## Chain IDs

Each environment has a unique chain ID used for signing transactions:

```typescript src/config/config.ts
export const CHAIN_IDS: Record<EGrvtEnvironment, number> = {
  [EGrvtEnvironment.DEV]: 327,
  [EGrvtEnvironment.STAGING]: 327,
  [EGrvtEnvironment.TESTNET]: 326,
  [EGrvtEnvironment.PRODUCTION]: 325,
};
```

Chain IDs are automatically included in EIP-712 signatures for transfers, withdrawals, and other signed operations. See the [Signing](/concepts/signing) documentation for more details.

## Domain Mapping

The SDK automatically maps environments to their corresponding domains:

```typescript src/client/GrvtBaseClient.ts
private getDomain(): string {
  switch (this.config.env) {
    case EGrvtEnvironment.PRODUCTION:
      return 'grvt.io';
    case EGrvtEnvironment.TESTNET:
      return 'testnet.grvt.io';
    case EGrvtEnvironment.STAGING:
      return 'staging.gravitymarkets.io';
    case EGrvtEnvironment.DEV:
      return 'dev.gravitymarkets.io';
    default:
      throw new Error(`Unknown environment: ${this.config.env}`);
  }
}
```

The edge base URL is constructed from the domain:

```typescript
this.edgeBaseUrl = `https://edge.${this.domain}`;
```

## Environment Configuration

When initializing the client, the environment is a required parameter:

```typescript src/config/config.ts
export interface IGrvtConfig {
  apiKey?: string;
  apiSecret?: string;
  tradingAccountId?: string;
  env: EGrvtEnvironment; // Required
}
```

## Best Practices

### Development Workflow

Follow this recommended workflow across environments:

<Steps>
  <Step title="Development">
    Use `EGrvtEnvironment.DEV` or `EGrvtEnvironment.TESTNET` for initial development and local testing.
  </Step>
  
  <Step title="Testing">
    Use `EGrvtEnvironment.TESTNET` for integration tests and CI/CD pipelines.
  </Step>
  
  <Step title="Staging">
    Use `EGrvtEnvironment.STAGING` for pre-production validation and UAT.
  </Step>
  
  <Step title="Production">
    Deploy to `EGrvtEnvironment.PRODUCTION` only after thorough testing in lower environments.
  </Step>
</Steps>

### Environment-Specific Credentials

Maintain separate API keys for each environment:

```typescript
// .env.development
GRVT_API_KEY=dev_key_...
GRVT_API_SECRET=dev_secret_...
GRVT_ENV=TESTNET

// .env.staging
GRVT_API_KEY=staging_key_...
GRVT_API_SECRET=staging_secret_...
GRVT_ENV=STAGING

// .env.production
GRVT_API_KEY=prod_key_...
GRVT_API_SECRET=prod_secret_...
GRVT_ENV=PRODUCTION
```

### Configuration Management

Use a configuration factory to manage environment-specific settings:

```typescript
function createClient(environment: string): GrvtClient {
  const env = environment === 'production' 
    ? EGrvtEnvironment.PRODUCTION 
    : EGrvtEnvironment.TESTNET;
    
  return new GrvtClient({
    apiKey: process.env.GRVT_API_KEY,
    apiSecret: process.env.GRVT_API_SECRET,
    tradingAccountId: process.env.GRVT_TRADING_ACCOUNT_ID,
    env
  });
}

const client = createClient(process.env.NODE_ENV);
```

## Switching Environments

To switch between environments, simply create a new client instance with the desired environment:

```typescript
// Switch from testnet to production
const testnetClient = new GrvtClient({
  apiKey: process.env.GRVT_TESTNET_API_KEY,
  apiSecret: process.env.GRVT_TESTNET_API_SECRET,
  env: EGrvtEnvironment.TESTNET
});

const productionClient = new GrvtClient({
  apiKey: process.env.GRVT_PRODUCTION_API_KEY,
  apiSecret: process.env.GRVT_PRODUCTION_API_SECRET,
  env: EGrvtEnvironment.PRODUCTION
});
```

<Warning>
Ensure you use the correct API keys for each environment. Using production keys in a test environment or vice versa will result in authentication errors.
</Warning>

## Environment Comparison

| Feature | DEV | STAGING | TESTNET | PRODUCTION |
|---------|-----|---------|---------|------------|
| **Stability** | Low | High | High | Highest |
| **Data Persistence** | Volatile | Stable | Stable | Permanent |
| **Chain ID** | 327 | 327 | 326 | 325 |
| **Real Assets** | No | No | No | Yes |
| **Recommended For** | Internal Dev | Pre-prod Testing | External Testing | Live Trading |
| **API Compatibility** | Experimental | Release Candidate | Stable | Stable |

## Troubleshooting

### Unknown Environment Error

If you receive an "Unknown environment" error:

```typescript
throw new Error(`Unknown environment: ${this.config.env}`);
```

Ensure you're using one of the valid `EGrvtEnvironment` enum values:

```typescript
// Correct
env: EGrvtEnvironment.TESTNET

// Incorrect
env: 'testnet' // This will fail
```

### Connection Issues

If you're having trouble connecting to an environment:

1. Verify the environment is operational
2. Check your API keys are valid for that environment
3. Ensure your network can reach the environment's domain
4. Check for any firewall or proxy restrictions
