# Quickstart Guide

Get up and running with the GRVT TypeScript SDK in just a few minutes. This guide will walk you through initializing the clients and making your first API calls.

## Prerequisites

Before you begin, make sure you have:

- Installed the SDK (see [Installation](/installation))
- GRVT API credentials (API key and API secret)
- Node.js 20 or higher

## REST API Client

The REST client provides access to account management, transfers, deposits, and withdrawals.

<Steps>
  <Step title="Import the SDK">
    First, import the necessary components from the SDK:

    ```typescript
    import {
      GrvtClient,
      ECurrency,
      ETransferType,
      EGrvtEnvironment
    } from '@grvt/sdk';
    ```
  </Step>

  <Step title="Initialize the client">
    Create a new `GrvtClient` instance with your API credentials:

    ```typescript
    const client = new GrvtClient({
      apiKey: 'your-api-key',
      apiSecret: 'your-api-secret',
      env: EGrvtEnvironment.DEV,
    });
    ```

    <Note>
      The `apiSecret` is optional but required for operations that need signing (transfers, withdrawals). If not provided, you must manually provide signatures.
    </Note>
  </Step>

  <Step title="Get account summary">
    Retrieve your funding account summary:

    ```typescript
    // Get funding account summary
    const accountSummary = await client.getFundingAccountSummary();
    console.log('Account summary:', accountSummary);

    // Get sub account summary
    const subAccountSummary = await client.getSubAccountSummary({
      sub_account_id: 'your-sub-account-id',
    });
    console.log('Sub-account summary:', subAccountSummary);
    ```
  </Step>

  <Step title="Execute a transfer">
    Transfer funds between accounts:

    ```typescript
    const transfer = await client.transfer({
      from_account_id: 'from-account-id',
      from_sub_account_id: 'from-sub-account-id',
      to_account_id: 'to-account-id',
      to_sub_account_id: 'to-sub-account-id',
      currency: ECurrency.USDT,
      num_tokens: '100',
      transfer_type: ETransferType.STANDARD,
    });
    console.log('Transfer result:', transfer);
    ```

    <Note>
      The SDK automatically signs the transfer request using your API secret. The signature field is optional and will be computed automatically if not provided.
    </Note>
  </Step>
</Steps>

## WebSocket Client

The WebSocket client enables real-time data streaming for transfers and other events.

<Steps>
  <Step title="Import the WebSocket client">
    Import the WebSocket client from the SDK:

    ```typescript
    import { GrvtWsClient, EGrvtEnvironment } from '@grvt/sdk';
    ```
  </Step>

  <Step title="Initialize and connect">
    Create a WebSocket client and establish a connection:

    ```typescript
    const wsClient = new GrvtWsClient({
      apiKey: 'your-api-key',
      env: EGrvtEnvironment.DEV,
    });

    // Connect to WebSocket
    await wsClient.connect();
    console.log('WebSocket connected!');
    ```
  </Step>

  <Step title="Subscribe to transfer history">
    Subscribe to real-time transfer updates:

    ```typescript
    wsClient.subscribeTransferHistory(
      'main-account-id',
      (data) => {
        console.log('Received transfer:', data);
      },
      'sub-account-id' // optional
    );
    ```

    <Note>
      The callback function receives transfer data in real-time as transfers occur on your account.
    </Note>
  </Step>

  <Step title="Handle disconnection">
    Disconnect when done:

    ```typescript
    // Disconnect when you're finished
    wsClient.disconnect();
    ```
  </Step>
</Steps>

## Complete Example

Here's a complete working example combining both REST and WebSocket clients:

```typescript
import {
  GrvtClient,
  GrvtWsClient,
  ECurrency,
  ETransferType,
  EGrvtEnvironment
} from '@grvt/sdk';

async function main() {
  // Initialize REST client
  const client = new GrvtClient({
    apiKey: 'your-api-key',
    apiSecret: 'your-api-secret',
    env: EGrvtEnvironment.DEV,
  });

  // Get account summary
  const accountSummary = await client.getFundingAccountSummary();
  console.log('Account summary:', accountSummary);

  // Initialize WebSocket client
  const wsClient = new GrvtWsClient({
    apiKey: 'your-api-key',
    env: EGrvtEnvironment.DEV,
  });

  // Connect and subscribe to transfers
  await wsClient.connect();
  wsClient.subscribeTransferHistory(
    'main-account-id',
    (data) => {
      console.log('Transfer received:', data);
    }
  );

  // Execute a transfer
  const transfer = await client.transfer({
    from_account_id: 'from-account-id',
    to_account_id: 'to-account-id',
    currency: ECurrency.USDT,
    num_tokens: '100',
    transfer_type: ETransferType.STANDARD,
  });
  console.log('Transfer executed:', transfer);

  // Keep the connection alive for 30 seconds to receive updates
  setTimeout(() => {
    wsClient.disconnect();
    console.log('Disconnected');
  }, 30000);
}

main().catch(console.error);
```

## Advanced Transfer with Metadata

For bridge deposits and withdrawals, you can provide metadata and custom signing options:

```typescript
import {
  GrvtClient,
  ECurrency,
  ETransferType,
  ETransferProvider,
  ETransferDirection,
  EChain,
  EGrvtEnvironment,
  ITransferMetadata,
  ISigningOptions
} from '@grvt/sdk';

const client = new GrvtClient({
  apiKey: 'your-api-key',
  apiSecret: 'your-api-secret',
  env: EGrvtEnvironment.DEV,
});

// Metadata for bridge deposit
const metadata: ITransferMetadata = {
  provider: ETransferProvider.RHINO,
  direction: ETransferDirection.DEPOSIT,
  chainid: EChain.TRON,
  endpoint: 'your-endpoint',
  provider_tx_id: 'tx-hash',
  provider_ref_id: 'commit-id',
};

// Custom signing options
const signingOptions: ISigningOptions = {
  nonce: 12345,
  expiration: '1746093221289693252'
};

const transfer = await client.transfer(
  {
    from_account_id: 'from-account-id',
    to_account_id: 'to-account-id',
    currency: ECurrency.USDT,
    num_tokens: '100',
    transfer_type: ETransferType.NON_NATIVE_BRIDGE_DEPOSIT,
  },
  metadata,
  signingOptions
);
```

<Warning>
  When using custom signing options, ensure that:
  - The nonce is non-negative
  - The expiration is within 30 days from the current time
</Warning>

## Query Historical Data

Retrieve transfer, deposit, and withdrawal history:

```typescript
// Get current server time
const currentTime = await client.getCurrentTime();
console.log('Server time:', currentTime); // milliseconds since epoch

// Query transfer history
// Note: both start_time and end_time are required and must be in nanoseconds
const transferHistory = await client.getTransferHistory({
  start_time: '1745600642000785050',
  end_time: '1758891778774100000'
});

// Query deposit history
const depositHistory = await client.getDepositHistory({
  start_time: '1745600642000785050',
  end_time: '1758891778774100000'
});

// Query withdrawal history
const withdrawalHistory = await client.getWithdrawalHistory({
  start_time: '1745600642000785050',
  end_time: '1758891778774100000'
});
```

<Note>
  All history query methods require both `start_time` and `end_time` parameters in nanoseconds. You can use the `getCurrentTime()` method to get the server time in milliseconds and convert it to nanoseconds.
</Note>

## WebSocket Features

The WebSocket client includes several powerful features:

### Automatic Reconnection

The client automatically reconnects with exponential backoff (up to 5 attempts) if the connection is lost:

```typescript
// Reconnection is automatic - no additional code needed
await wsClient.connect();
```

### Connection Monitoring

The client monitors the connection and reconnects if no messages are received within 5 seconds:

```typescript
// Connection monitoring is automatic
// Reconnects if no messages received for 5 seconds
```

### Manual Resubscription

<Warning>
  Subscriptions are NOT automatically restored after reconnection. You must manually resubscribe after a reconnection event.
</Warning>
