## Overview

The `GrvtClient` class is the primary client for interacting with GRVT's REST API endpoints. It provides methods for account management, transfers, deposits, withdrawals, and querying transaction history.

## Constructor

```typescript
new GrvtClient(config: IGrvtConfig)
```

### Parameters

<ParamField path="config" type="IGrvtConfig" required>
  Configuration object for the client
  
  <Expandable title="IGrvtConfig properties">
    <ParamField path="apiKey" type="string">
      Your GRVT API key for authentication
    </ParamField>
    
    <ParamField path="apiSecret" type="string">
      Your GRVT API secret for signing operations (required for withdraw, transfer, and deposit operations)
    </ParamField>
    
    <ParamField path="tradingAccountId" type="string">
      Your trading account ID
    </ParamField>
    
    <ParamField path="env" type="EGrvtEnvironment" required>
      The environment to connect to: `PRODUCTION`, `TESTNET`, `STAGING`, or `DEV`
    </ParamField>
  </Expandable>
</ParamField>

### Example

```typescript
import { GrvtClient, EGrvtEnvironment } from '@grvt/client';

const client = new GrvtClient({
  apiKey: 'your-api-key',
  apiSecret: 'your-api-secret',
  tradingAccountId: 'your-account-id',
  env: EGrvtEnvironment.TESTNET
});
```

## Authentication

The client automatically handles authentication using your API key. For operations that require signing (transfers, withdrawals, deposits), you must provide an `apiSecret` in the configuration.

<Info>
  All methods return Promises and should be used with `async/await` or `.then()/.catch()`.
</Info>

## Error Handling

```typescript
try {
  const summary = await client.getFundingAccountSummary();
  console.log(summary);
} catch (error) {
  console.error('API error:', error);
}
```
