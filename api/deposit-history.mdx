## Overview

The `getDepositHistory()` method retrieves historical deposit records for your account within a specified time range. This includes deposits made through L1 bridges or direct transfers (for Arbitrum).

## Method Signature

```typescript
getDepositHistory(
  request: IApiDepositHistoryRequest
): Promise<IApiDepositHistoryResponse>
```

## Parameters

<ParamField path="request" type="IApiDepositHistoryRequest" required>
  The deposit history query request object with the following properties:
</ParamField>

<ParamField path="request.start_time" type="string" required>
  Start timestamp in nanoseconds (Unix epoch). Only deposits with event_time greater than or equal to start_time will be returned.
</ParamField>

<ParamField path="request.end_time" type="string" required>
  End timestamp in nanoseconds (Unix epoch). Only deposits with event_time less than or equal to end_time will be returned.
</ParamField>

<ParamField path="request.sub_account_id" type="string">
  Optional sub-account ID to filter deposits. If not provided, returns deposits for all sub-accounts.
</ParamField>

<ParamField path="request.currency" type="ECurrency">
  Optional currency filter. Filter deposits by specific currency (e.g., USDT, USDC, BTC, ETH).
</ParamField>

<ParamField path="request.limit" type="number">
  Maximum number of records to return. Use for pagination.
</ParamField>

<ParamField path="request.cursor" type="string">
  Pagination cursor for fetching subsequent pages of results.
</ParamField>

## Return Value

<ResponseField name="result" type="IApiDepositHistoryResponse" required>
  The deposit history response object containing:
</ResponseField>

<ResponseField name="result.deposits" type="array">
  Array of deposit records matching the query criteria. Each deposit includes:
  - Deposit ID
  - Account ID
  - Sub-account ID
  - Currency
  - Amount
  - Deposit status
  - Event timestamp
  - Transaction hash (if applicable)
  - Chain information
</ResponseField>

<ResponseField name="result.next_cursor" type="string">
  Cursor for fetching the next page of results. If null, no more results are available.
</ResponseField>

## Timestamp Format

<Info>
  All timestamps in deposit history queries are in **nanoseconds** since Unix epoch (January 1, 1970 00:00:00 UTC). This provides high precision for time-based queries.
  
  To convert from milliseconds to nanoseconds, multiply by 1,000,000:
  ```typescript
  const milliseconds = Date.now();
  const nanoseconds = (milliseconds * 1000000).toString();
  ```
</Info>

## Pagination

For large result sets, use the pagination support:

1. Specify a `limit` in your initial request
2. Check the `next_cursor` field in the response
3. If `next_cursor` is not null, make another request with the cursor to fetch the next page

## Example

```typescript
import { GrvtClient, EGrvtEnvironment, ECurrency } from '@grvt/sdk';

// Initialize the client
const client = new GrvtClient({
  apiKey: 'your-api-key',
  apiSecret: 'your-api-secret',
  env: EGrvtEnvironment.DEV,
});

// Query deposit history
// Note: both start_time and end_time are required for history queries
const depositHistory = await client.getDepositHistory({
  start_time: '1745600642000785050', // timestamp in nanoseconds
  end_time: '17588917787741000000' // timestamp in nanoseconds
});
// You can filter more & do pagination with this query if needed, please take a look at the request interface to get more details

// Example with pagination and filters
const filteredHistory = await client.getDepositHistory({
  start_time: '1745600642000785050',
  end_time: '17588917787741000000',
  sub_account_id: 'sub-account-123',
  limit: 50,
  currency: ECurrency.USDT
});

// Fetch next page if available
if (filteredHistory.next_cursor) {
  const nextPage = await client.getDepositHistory({
    start_time: '1745600642000785050',
    end_time: '17588917787741000000',
    cursor: filteredHistory.next_cursor
  });
}
```

## Validation

<Warning>
  Both `start_time` and `end_time` are **required** parameters. The method will throw an error if either is missing:
  - "startTime is required"
  - "endTime is required"
</Warning>
